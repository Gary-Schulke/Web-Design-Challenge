Summary: Latitude vs. X

The Latitude project compares maximum temperature, humidity, winds speed, and cloudiness based on the earth's latitude. All samples are taken using the same date. Sample sites were selected by randomly generating latitude and longitude pairs then using the citypy library to select the nearest city. This method of finding cities generally found approximately 620 cities from the 1500 latitude-longitude pairs randomly generated. Calling the api directly using the generated latitudes and longitudes was attempted but produced an excessive amount of empty results. The strength of using citypy is that it greatly reduces queries to the api that return nothing. The weakness is that not all latitudes are represented equally. This could be caused by citypy if its locations favor the northern hemisphere. There could also be more northern hemisphere reporting locations on openweathermap.org.

The openweathermap.org public api was used to aquire the data. Five-hundred samples were taken and plotted. Calls to the api that returned no results were omitted. Very little data munging was required because the returned data was clean. Pandas was used to put the data in a data frames. Matplotlib was used to create the virtualizations.

The data for these charts was taken on November 17, 2019.
